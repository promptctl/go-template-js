# Conformance fixtures

Each subdirectory is one fixture: `template.tmpl` + optional `scope.json` +
exactly one expected-outcome file.

## Expected-outcome variants

A fixture has **exactly one** of:

- `expected.txt` — byte-equality parity with Go's `text/template + sprig`.
  Generated by `pnpm conformance:regen`. Asserted by the string-mode harness
  (`test/conformance/string-mode.test.ts`).
- `expected-fragments.json` — structural equality on the typed-T fragment
  list. Asserted by `test/conformance/typed-fragment.test.ts`. JS-only;
  the Go generator skips these.
- `expected-error.json` — error parity: the engine must throw the named
  error. Shape: `{ "kind": "TypeMismatchError", "funcName": "...", "argIndex": N }`.
  Asserted by `test/conformance/error-parity.test.ts`. JS-only; the Go
  generator skips these.

There is no skip-list and no divergence file. A fixture either succeeds
with one of the parity contracts above or it does not exist.

## Naming convention

`<category>-<short-description>/` where category is one of:

- `controlflow` — if / range / with / define / template / block constructs
- `builtin` — Go template runtime built-ins (eq/lt/and/len/index/slice/printf/call/...)
- `sprig` — Sprig-subset functions (one per category, plus highlights)
- `edge` — whitespace trim, comments, deeply nested, etc.
- `typed` — generic-T fragment behaviour (uses `tagAs`)
- `negative` — error-parity fixtures asserting the no-silent-flatten guard fires

## Adding a fixture

1. Create the subdir with `template.tmpl`.
2. Add `scope.json` if the template needs scope (omit for nil-scope templates).
3. Add exactly one expected-outcome file:
   - For Go-parity fixtures: run `pnpm conformance:regen` to generate `expected.txt`.
   - For typed-fragment fixtures: hand-author `expected-fragments.json`.
   - For error-parity fixtures: hand-author `expected-error.json`.
4. The matching harness under `test/conformance/` will pick the fixture up
   automatically — no test code change required.

## Avoid

Patterns that won't byte-match Go:

- Regex with lookbehind or Unicode property escapes.
- Float formatting at extreme precision.
- Bare `range` over a map (Go's map iteration is randomized; JS object
  iteration is insertion-ordered). Sort the keys first if you need it.
